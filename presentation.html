Contexte et objectif

Le problème

Futurisys souhaite anticiper les départs volontaires de ses employés pour réduire les coûts de recrutement et de formation.

Un modèle LightGBM a été entraîné sur des données RH (SIRH, évaluations, sondage de satisfaction).

L'objectif de ce projet

Mettre ce modèle en production :

  • Exposer le modèle via une API REST
  • Journaliser chaque prédiction en base de données
  • Automatiser les tests et le déploiement CI/CD
  • Rendre l'API accessible en ligne via Hugging Face

Pipeline CI/CD — GitHub Actions

① Job : test

  • S'exécute à chaque push sur main
  • Installe les dépendances
  • Lance pytest tests/
  • Bloque le déploiement si un test échoue

② Job : deploy

  • Ne s'exécute que si les tests passent
  • Copie le code vers Hugging Face Spaces
  • Utilise HF_TOKEN (secret GitHub chiffré)

Gestion des secrets

  • HF_TOKEN — stocké dans les secrets GitHub
  • DATABASE_URL — variable d'environnement

API FastAPI — Démonstration

Endpoints

  • GET / — vérification que l'API tourne
